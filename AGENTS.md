# ロボットコージョー（ガチャ）案内（プログラムできない人向け）

このフォルダは、ブラウザだけで動く「ロボットコージョー ガチャ」のプロジェクトです（React + Vite）。
文章・ボタン名・ガチャの中身（画像/音）を差し替えるのが主な作業になります。

## どのファイルを触ればいい？

- 画面の中身（文章/ボタン/動き）: `src/App.jsx`
  - 先頭の `FILES = [...]` が「ガチャで出る画像の一覧」です
- 見た目（色/レイアウト）: `src/styles.css`
- 画像（ガチャの素材）: `public/images/`
- BGM: `public/bgm/bgm1.mp3`
- 効果音: `public/se/se1.mp3` から `public/se/se7.mp3`
- 注意: 似た名前の `images/` や `dist/images/` ではなく、基本は `public/` の中を編集します

## 触らない（編集しない）もの

- `dist/`（公開用の完成品。`npm run build` で毎回作り直されます）
- `node_modules/`（ライブラリ。基本触りません）

## 動かし方（プレビュー）

このプロジェクトは「ファイルをダブルクリックで開く」だと動きません。ローカルサーバーが必要です。

1. 初回だけ: `npm install`
2. 起動: `npm run dev`
3. ブラウザで開く: 表示されたURL（だいたい `http://localhost:5173/`）
4. 止める: ターミナルで `Ctrl + C`

完成品の確認（`dist/` を使う）:
- `npm run build`
- `npm run preview`（表示されたURLを開く）

## 公開（アップロード）するとき

- `npm run build` を実行して `dist/` を更新します
- サーバーにアップするのは `dist/` の中身です（`src/` や `node_modules/` は不要）
- GitHub Pages など「サブフォルダ配下」で公開する場合、`vite.config.js` の `base` 設定が必要になることがあります（分からなければAIに頼んでください）

## 遊び方（サイトを見る人向け）

- 「ガチャ」: 1回引く
- 「足立図鑑」: 全カード一覧（未入手は `???`、入手済みはクリックで拡大）
- 「所持カード一覧」: 入手したカード（「組み立てる」中はドラッグで貼れます）
- 「組み立てる」: 所持カードを貼り付けて配置（選択して `Delete/Backspace` で削除）
- BGMはブラウザの都合で、最初のクリック後に鳴り始めます

## よくある変更（AIに頼む用テンプレ）

そのままコピペして、`【】`の中だけ書き換えてください。

- タイトルを変えたい  
  「`index.html` の `<title>` と、`src/App.jsx` の `<h1>` を【新しいタイトル】に変更して」

- 下の説明文を変えたい（画面いちばん下の文章）  
  「`src/App.jsx` の末尾にある `<p className="small">` の文章を【新しい文章】に変更して」

- ガチャに出る画像を追加したい  
  「`public/images/` に【ファイル名.png】を追加して、`src/App.jsx` 先頭の `FILES` 配列にも同じファイル名を追加して」
  - 画面に出る名前は「ファイル名から `.png` を外したもの」です

- ガチャに出る画像を減らしたい  
  「`src/App.jsx` の `FILES` から【消したいファイル名.png】を削除して（必要なら `public/images/` の画像も削除して）」

- BGMを差し替えたい（今のままファイル名固定でOK）  
  「`public/bgm/bgm1.mp3` を【新しいBGM】に差し替えて」

- 効果音を差し替えたい  
  「`public/se/se3.mp3`（など）を【新しい効果音】に差し替えて」

- 「全キャラ入手」ボタンを消したい（デバッグ用っぽいボタン）  
  「`src/App.jsx` の `全キャラ入手` ボタンを非表示/削除して」

- 収集状況がリロードで消えるのを直したい  
  「`src/App.jsx` にある『localStorage を消している useEffect』をやめて、引いた結果が保存されるようにして」
